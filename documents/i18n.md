# i18n (Çoklu Dil)

## Genel yapı

GitDeep frontend'i `next-intl` ile `en` (İngilizce) ve `tr` (Türkçe) dillerini
destekler. URL yapısı locale-prefixed'dir: `/en/...` ve `/tr/...`
(`frontend/src/app/[locale]/` segment yapısı).

- Yönlendirme yapılandırması: `frontend/src/i18n/routing.ts`
- Locale-farkında `Link`/`useRouter`/`usePathname`: `frontend/src/i18n/navigation.ts`
  (`createNavigation(routing)` ile üretilir — her zaman bunlar kullanılır,
  çıplak `next/navigation` değil, çünkü path'e locale önekini otomatik ekler).
- Sunucu tarafı locale çözümleme: `frontend/src/i18n/request.ts`

## Çeviri dosyaları

`frontend/messages/en.json` ve `frontend/messages/tr.json` — düz, iç içe JSON
nesneleri. Her üst seviye anahtar bir "namespace" (örn. `landing`, `nav`,
`auth`, `dashboard`, `analyze`, `job`, `report`, `history`, `billing`).

Bileşen içinde kullanım:
```tsx
const t = useTranslations("analyze");
t("title")                          // basit anahtar
t("balanceChange", { before, after })  // {placeholder} interpolasyonu
```

## KRİTİK KURAL: iki dosya her zaman birlikte güncellenir

Yeni bir UI metni eklerken **hem `en.json` hem `tr.json`'a aynı anahtar aynı
yapıda eklenmelidir**. Bu proje boyunca hiçbir zaman ihlal edilmemiş bir
kuraldır — iki dosya arasında anahtar kümesi her zaman birebir aynıdır (fark
varsa build zamanı hata vermez ama eksik dilde ham anahtar adı görünür ya da
`next-intl` uyarısı basılır).

Doğrulama (herhangi bir anahtar farkı olup olmadığını kontrol etmek için):
```bash
python3 -c "
import json
en = json.load(open('frontend/messages/en.json'))
tr = json.load(open('frontend/messages/tr.json'))
def flatten(d, prefix=''):
    out = {}
    for k, v in d.items():
        key = f'{prefix}.{k}' if prefix else k
        out.update(flatten(v, key) if isinstance(v, dict) else {key: v})
    return out
en_keys, tr_keys = set(flatten(en)), set(flatten(tr))
print('sadece en:', en_keys - tr_keys)
print('sadece tr:', tr_keys - en_keys)
"
```

## Mevcut namespace'ler (özet)

| Namespace | Kullanıldığı yer |
|---|---|
| `landing` | Ana sayfa (pazarlama). |
| `nav` | Global `Header` bileşeni (login/register/dashboard/pricing/logout). |
| `common` | Paylaşılan küçük metinler (`credits`, `loading`). |
| `auth` | Login/register formu, e-posta doğrulama sayfası. |
| `dashboard` | Panel sayfası (bakiye, kredi geçmişi, e-posta doğrulama banner'ı). |
| `analyze` | Repo linki gir → teklif akışı. |
| `job` | Analiz canlı ilerleme sayfası. |
| `report` | Rapor görünümü (bölüm başlıkları, PDF indirme). |
| `history` | Analiz geçmişi tablosu. |
| `billing` | Kredi satın alma sayfası. |

## Backend tarafında dil

Kullanıcının `locale` alanı (`tr`/`en`, kayıt sırasında seçilir) hem e-posta
şablonlarında (`email_en.html`/`email_tr.html`) hem de LLM'in rapor dilinde
kullanılır — Claude, README'den tespit ettiği dile göre (`detected_language`)
raporu otomatik olarak Türkçe veya İngilizce üretir (bkz.
[mimari.md](mimari.md#analiz-pipeline)).
